âœ… Thread Management
   - start_camera_processing()    Launch background thread
   - stop_camera_processing()     Stop processing
   - get_camera_status()          Check processing status
   - Thread-safe operations

PIPELINE:
1. API receives camera IP
2. Create/get Camera record in database
3. Launch background thread (non-blocking)
4. Load YOLOv8 model (~200MB, cached)
5. Open RTSP stream using OpenCV
6. For each frame:
   - Read frame from stream
   - Resize for performance
   - Run YOLOv8 inference
   - Filter for "person" class (class_id=0)
   - Count detections
   - Accumulate statistics
7. Every 60 seconds:
   - Calculate maximum count in interval
   - Compute average inference time
   - Save CameraCount record
   - Reset for next interval
8. Handle errors gracefully:
   - Stream disconnect â†’ auto-retry
   - Timeout â†’ mark offline
   - Inference error â†’ log & continue

PERFORMANCE:
- Model: YOLOv8n (nano) ~5ms inference/frame
- FPS: ~30 with mid-range GPU
- Memory: ~500MB total
- Threading: Non-blocking HTTP requests
- Reliability: Auto-reconnect on failure
